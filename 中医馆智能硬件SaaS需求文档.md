## 二、目标客户画像

### 2.1 单店中医馆特征

| 维度 | 描述 |
|-----|------|
| **规模** | 1-3名中医师，1-2间诊室 |
| **痛点** | 缺乏现代化诊疗工具、患者管理粗放、复诊率低 |
| **预算** | 1-2万/年的IT投入 |
| **决策人** | 馆长（通常是老中医） |

|-----|---------|---------|

| 行业协会（中华中医药学会） | ¥2000/年 | 中 |
| 行业社群（中医馆主微信群） | ¥500 | 短 |
| 阿里巴巴/慧聪网店铺 | ¥3000/年 | 长 |

## 三、产品功能模块

### 3.1 智能硬件（需选型采购）

| 硬件类型 | 功能 | 预估成本 | 销售价 |
|---------|------|---------|-------|
| **体质检测一体机** | 舌诊+面诊+问诊，AI体质判定 | ¥3000 | ¥5000 |
| **便携脉象仪** | 寸关尺三部脉象采集 | ¥1500 | ¥3000 |
| **智能艾灸仪** | 穴位定位+温度控制 | ¥2000 | ¥4000 |

> 建议：MVP阶段先做**体质检测一体机**，功能集成度高、单价适中

### 3.2 SaaS功能清单

#### P0（必须）

| 模块 | 功能 | 描述 |
|-----|------|-----|
| 患者档案 | 建档管理 | 基本信息、病史、体质记录 |
| 预约挂号 | 在线预约 | 日期、医师、时段管理 |
| 诊疗记录 | 电子病历 | 诊断、处方、医嘱 |
| 体质检测 | 硬件对接 | 舌诊/脉象AI分析 |
| 随访管理 | 复诊提醒 | 微信消息推送 |

#### P1（期望）

| 模块 | 功能 | 描述 |
|-----|------|-----|
| 会员管理 | 储值/疗程卡 | 预付费管理 |
| 药房对接 | 处方流转 | 对接中药房/代煎 |
| 数据报表 | 经营分析 | 营收、客户、复诊率 |
| 商城 | 养生产品 | 膏方、艾灸条等 |

### 3.3 流量变现

| 功能 | 描述 | 收益模式 |
|-----|------|---------|
| 线上问诊 | 图文/视频咨询 | 诊疗费抽佣5% |
| 体质商城 | 推荐养生产品 | 商品佣金10-30% |
| 复诊续方 | 线上复诊购药 | 诊疗费抽佣5% |

## 四、技术架构

### 4.1 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端（微信小程序）                       │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │患者端小程序  │  │医师端小程序 │  │管理后台（Web）   │ │
│  │预约/问诊/商城│  │诊疗/病历    │  │患者/预约/报表   │ │
│  └─────────────┘  └─────────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                      后端服务（Java云服务器）                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────┐│
│  │用户服务  │ │预约服务  │ │诊疗服务  │ │硬件通信服务  ││
│  │支付服务  │ │病历服务  │ │AI分析服务│ │设备管理      ││
│  └──────────┘ └──────────┘ └──────────┘ └──────────────┘│
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                      硬件设备                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │
│  │体质检测一体机│  │便携脉象仪    │  │智能艾灸仪   │   │
│  │(MQTT/HTTP)   │  │(蓝牙/USB)    │  │(WiFi)        │   │
│  └──────────────┘  └──────────────┘  └──────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 硬件对接方案

| 设备类型 | 通信方式 | 对接难度 | 建议 |
|---------|---------|---------|------|
| 体质检测一体机 | HTTP REST API | ⭐⭐ | 主流厂商有开放API |
| 便携脉象仪 | 蓝牙BLE | ⭐⭐⭐ | 需要定制开发 |
| 智能艾灸仪 | WiFi/MQTT | ⭐⭐ | 工业协议为主 |

> MVP建议：先对接**有成熟API的体质检测一体机**，跳过硬件研发，只做软件集成

### 4.3 推荐硬件供应商

| 供应商 | 产品 | 特点 | API开放 |
|-------|------|-----|--------|
| 上海中医药大学 | 舌面诊仪 | 高校背书 | 有 |
| 某科技公司 | 综合体质检测仪 | 一体化 | 有 |
| 某医疗 | 脉象仪 | 医疗级 | 需定制 |

## 七、开发计划（3个月）

```
第1阶段：产品调研（2周）
  Week 1    调研3-5家硬件供应商，确定合作方
  Week 1    竞品SaaS分析（看其他中医馆软件）
  Week 2    整理客户痛点需求

第2阶段：MVP开发（6周）
  Week 3-4  硬件API对接Demo
  Week 4-5  患者端小程序（预约+问诊）
  Week 5-6  医师端小程序（诊疗+病历）
  Week 6    管理后台（基础功能）

第3阶段：客户验证（2周）
  Week 7    找2-3家中医馆试用
  Week 7    收集反馈，修复bug

第4阶段：推广获客（2周+）
  Week 8    参加1次行业展会
  Week 8+   行业社群推广
```
